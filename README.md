# fastest

> **Warning:** This README was generated by AI from the project source files and headers.

A macro-based C testing framework built around a global test scheduler. Tests register themselves before `main()` via `__attribute__((constructor))` and can be run from C or Python. Nanosecond-precision timing and memory tracking included; zero external dependencies for the core library.

## Integration

### C-only

Copy the library and headers into your project, then compile:

```bash
cp build/libfastest.a /your/project/lib/
cp -r include/fastest /your/project/include/

gcc -std=c11 -o tests tests.c -Iinclude -Llib -lfastest
```

Add `-DDEBUG` to enable `DEBUG_PRINTF` output.

### Demo project layout

The shared Makefile expects this layout inside your demo directory:

```
your-demo/
├── src/              # your .c source files
├── vendor/
│   └── fastest/      # symlink or copy of the fastest repo root
├── Makefile          # symlink to fastest/demo/shared_Makefile
└── build_wheel.sh    # symlink to fastest/demo/shared_build_wheel.sh
```

The symlink approach used in the demos:

```bash
ln -s /path/to/fastest/demo/shared_Makefile Makefile
ln -s /path/to/fastest/demo/shared_build_wheel.sh build_wheel.sh
mkdir -p vendor
ln -s /path/to/fastest vendor/fastest
```

The Makefile compiles all `.c` files under `src/` as PIC objects and archives them into `build/release/lib<demo-name>.a`. `FASTEST_HOME` defaults to `vendor/fastest`.

```bash
make              # release build → build/release/lib<name>.a
make BUILD=debug  # debug build with -g -O0 -DDEBUG
make clean
```

## Testing Modes

fastest provides four ways to define tests.

### 1. QUICKTEST — Expression-based

Tests a boolean expression inline. Timing is measured around the expression. Does **not** register in the scheduler — result is printed immediately.

```c
FASTEST_QUICKTEST(name, expr, flags)
```

```c
FASTEST_QUICKTEST("add 2+3==5", add(2, 3) == 5,
    FASTEST_ASSERT_EQ | FASTEST_FAIL_ERROR | FASTEST_TIME_NS);
```

### 2. CUSTOMTEST — Scheduler-registered function

Registers a named test function in the global scheduler via `__attribute__((constructor))`. The function receives `FASTEST_TestOutput_t *out` and sets flags and status manually.

```c
FASTEST_CUSTOMTEST(name, preload_flags, test_func, callback)
```

```c
void my_test(FASTEST_TestOutput_t *out) {
    out->test_flags  |= FASTEST_ASSERT_EQ;
    out->exit_status |= add(2, 3) == 5 ? FASTEST_SUCCESS : FASTEST_ERROR_ASSERT;
}

FASTEST_CUSTOMTEST("my test", FASTEST_TIME_NS | FASTEST_FAIL_ERROR, my_test, NULL);
```

`preload_flags` are merged into `out->test_flags` before the test runs. `callback` is an optional `void (*)(FASTEST_TestOutput_t *)` called after execution; pass `NULL` to skip.

### 3. CUSTOMTEST\_INLINE — Inline body, external callback

Test body is written inline; callback is a separate named function.

```c
FASTEST_CUSTOMTEST_INLINE(name, preload_flags, callback, { body })
```

```c
void after(FASTEST_TestOutput_t *out) {
    DEBUG_PRINTF("%s done", out->test_name);
}

FASTEST_CUSTOMTEST_INLINE("inline add", FASTEST_TIME_NS | FASTEST_FAIL_ERROR,
    after,
    {
        out->test_flags  |= FASTEST_ASSERT_EQ;
        out->exit_status |= add(2, 3) == 5 ? FASTEST_SUCCESS : FASTEST_ERROR_ASSERT;
    }
)
```

### 4. CUSTOMTEST\_DINLINE — Fully inline

Both test body and callback body are written inline. Most compact form.

```c
FASTEST_CUSTOMTEST_DINLINE(name, preload_flags, { callback_body }, { test_body })
```

```c
FASTEST_CUSTOMTEST_DINLINE("dinline add",
    FASTEST_TIME_NS | FASTEST_FAIL_ERROR,
    {
        DEBUG_PRINTF("%s done, assert failed: %d",
            out->test_name,
            (out->exit_status & FASTEST_ERROR_ASSERT) > 0);
    },
    {
        out->test_flags  |= FASTEST_ASSERT_EQ;
        out->exit_status |= add(2, 3) == 5 ? FASTEST_SUCCESS : FASTEST_ERROR_ASSERT;
    }
)
```

### Running Scheduled Tests from C

Tests registered with CUSTOMTEST / INLINE / DINLINE are stored in the global scheduler and can be run from anywhere:

```c
#include "fastest/test_list.h"

int main(void) {
    FASTEST_List_t *list;
    FASTEST_list_get_instance(&list);

    FASTEST_list_exec_name(list, "my test");  // run by name
    FASTEST_list_exec(list, 0);               // run by index
    return 0;
}
```

Timing and logging are handled automatically by the scheduler based on `preload_flags`.

## Flags Reference

Flags are `uint64_t` bitmasks combined with `|`.

### Assertion

| Flag | Assertion |
|------|-----------|
| `FASTEST_ASSERT_EQ` | `==` |
| `FASTEST_ASSERT_NEQ` | `!=` |
| `FASTEST_ASSERT_GT` | `>` |
| `FASTEST_ASSERT_GE` | `>=` |
| `FASTEST_ASSERT_LT` | `<` |
| `FASTEST_ASSERT_LE` | `<=` |

### Failure reporting

| Flag | Output style |
|------|-------------|
| `FASTEST_FAIL_ERROR` | Red `[FASTEST ERROR]` |
| `FASTEST_FAIL_WARNING` | Yellow `[FASTEST WARNING]` |
| `FASTEST_FAIL_LOG` | Blue `[FASTEST LOG]` |

### Timing

| Flag | Unit |
|------|------|
| `FASTEST_TIME_NS` | nanoseconds |
| `FASTEST_TIME_US` | microseconds |
| `FASTEST_TIME_MS` | milliseconds |
| `FASTEST_TIME_S` | seconds |

### Misc

| Flag | Effect |
|------|--------|
| `FASTEST_MEM_TRACK` | Track allocation/deallocation, report leaks |
| `FASTEST_DEFAULT_LOG` | Enable `FASTEST_print_result` output |

### Exit status codes

Set on `out->exit_status` inside test functions.

| Code | Meaning |
|------|---------|
| `FASTEST_SUCCESS` | Test passed |
| `FASTEST_ERROR_ASSERT` | Assertion failed |
| `FASTEST_ERROR_MEMORY` | Memory leak detected |
| `FASTEST_ERROR_TIMEOUT` | Test exceeded time limit |
| `FASTEST_ERROR_EXCEPTION` | Signal or fatal error |
| `FASTEST_ERROR_UNEXPECTED` | Unexpected result |
| `FASTEST_ERROR_RESOURCE` | I/O or resource failure |
| `FASTEST_ERROR_MPI` | MPI failure |
| `FASTEST_ERROR_OMP` | OpenMP failure |
| `FASTEST_ERROR_CUDA` | CUDA failure |
| `FASTEST_ERROR_INTERNAL` | Framework misuse |
| `FASTEST_ERROR_COLLISION` | Duplicate test name |
| `FASTEST_ERROR_NOT_FOUND` | Test not in scheduler |

## Python Bindings

### Extension module (pybind11)

The `bindings/` directory contains a pybind11 extension that wraps the C scheduler. It is compiled per-project via `build_wheel.sh` and produces a module named after the demo (e.g. `showcase`).

From your demo directory:

```bash
./build_wheel.sh
```

The script sets `FASTEST_HOME`, `FASTEST_USER_LIB` (your compiled `.a`), and `FASTEST_MODULE_NAME` (defaults to the directory name), then runs `pip install -e .` from `bindings/`. It also generates a `.pyi` stub for IDE autocompletion.

#### Low-level extension API

```python
import showcase

showcase.get_tests()                  # list of all test dicts
showcase.get_test("my test")          # single test dict by name
showcase.get_subtests("suite")        # tests whose names start with "suite/"
showcase.run_test("my test")          # execute and update scheduler state

showcase.tests.my_test                # "my test"  (name constant)
showcase.tests.Dinline                # "Dinline"
```

Each test dict contains:

```python
{
    "test_name":    str,
    "status":       str,   # "Pending" | "Running" | "Executed"
    "test_flags":   int,
    "exit_status":  int,
    "allocation":   int,
    "deallocation": int,
    "time_ns":      int,
}
```

#### Stub (`.pyi`)

```python
from typing import Any

def get_tests() -> list[dict[str, Any]]: ...
def get_test(name: str) -> dict[str, Any]: ...
def get_subtests(prefix: str) -> list[dict[str, Any]]: ...
def run_test(name: str) -> None: ...

class tests:
    Dinline: str
    custom_test: str
    inline: str
```

## Python Package

`fastest_py/` is a pure-Python package (`import fastest`) that sits on top of the compiled extension and provides high-level orchestration: running pools of tests, collecting timing statistics, comparing pools, and rendering reports.

### Installation

```bash
cd fastest_py
pip install -e .
```

### Setting a backend

The package needs to know which compiled extension to use. Set `FASTEST_PROJECT` to the module name:

```bash
FASTEST_PROJECT=showcase python main.py
```

Or set it programmatically:

```python
import fastest
import showcase

fastest.default_runner.set_backend(showcase)
```

### Module facade

`fastest` delegates unknown attribute access to `default_runner`, so you can call runner methods directly on the module:

```python
import fastest

fastest.run_test("my test")
fastest.get_tests()
fastest.run_all()
fastest.tests.Dinline           # test name constant from the backend
```

### Runner

`Runner` is the main entry point. When `FASTEST_PROJECT` is set it auto-detects and loads the backend.

```python
from fastest import Runner
import showcase

runner = Runner(showcase)
runner.verbose = True           # print each test name as it runs
runner.run_all()
```

### Pools

A `Pool` is a named list of test names to benchmark together.

```python
# Explicit list
pool1 = fastest.pool("pool1", fastest.tests.Dinline, fastest.tests.custom_test)

# All tests sharing a prefix
pool2 = fastest.pool_from_prefix("suite")
```

### Comparing pools

```python
result = fastest.compare(pool1, pool2, n_repeats=5)
result.report()          # print to stdout
result.save("out.json")  # serialise to JSON
```

`compare` runs every test in every pool `n_repeats` times, collects `time_ns` samples, and returns a `CompareResult` with per-test `Stats` (mean, stddev, min, max, median) and a cross-pool paired comparison table.

### flags module

`fastest.flags` mirrors every bitmask from `tests.h` and provides string helpers:

```python
from fastest import flags

flags.ASSERT_EQ          # 0x1
flags.TIME_NS            # 0x1000
flags.SUCCESS            # 0x1
flags.ERROR_ASSERT       # 0x100

flags.strexit(exit_status, test_flags)    # "SUCCESS" | "ASSERT_EQ failed" | ...
flags.strassert(test_flags)               # "ASSERT_EQ" | "ASSERT_GT" | ...
flags.strtime(test_flags)                 # "ns" | "µs" | "ms" | "s"
flags.symbol(exit_status)                # "✓" | "✗" | "~" | "?"
flags.passed(exit_status)                # bool
flags.strleak(allocation, deallocation)  # "leak: 64 bytes (...)" | None
flags.strtest(test_dict)                 # formatted one-liner for a test dict
```

## Examples

### Minimal C test

```c
#include "fastest/quick_tests.h"
#include "fastest/tests.h"

int add(int a, int b) { return a + b; }

int main(void) {
    FASTEST_QUICKTEST("add 2+3==5", add(2, 3) == 5,
        FASTEST_ASSERT_EQ | FASTEST_FAIL_ERROR | FASTEST_TIME_NS);
    return 0;
}
```

### All four modes

```c
#include "fastest/custom_tests.h"
#include "fastest/quick_tests.h"
#include "fastest/tests.h"

int add(int a, int b) { return a + b; }

void inline_callback(FASTEST_TestOutput_t *out) {
    DEBUG_PRINTF("%s executed, assert failed: %d",
        out->test_name, (out->exit_status & FASTEST_ERROR_ASSERT) > 0L);
}

FASTEST_CUSTOMTEST_INLINE("inline", FASTEST_TIME_NS | FASTEST_FAIL_ERROR,
    inline_callback,
    {
        out->test_flags  |= FASTEST_ASSERT_EQ;
        out->exit_status |= add(2, 3) == 5 ? FASTEST_SUCCESS : FASTEST_ERROR_ASSERT;
    }
)

FASTEST_CUSTOMTEST_DINLINE("Dinline", FASTEST_TIME_NS | FASTEST_FAIL_ERROR,
    {
        DEBUG_PRINTF("%s executed, assert failed: %d",
            out->test_name, (out->exit_status & FASTEST_ERROR_ASSERT) > 0L);
    },
    {
        out->test_flags  |= FASTEST_ASSERT_EQ;
        out->exit_status |= add(2, 3) == 5 ? FASTEST_SUCCESS : FASTEST_ERROR_ASSERT;
    }
)

void my_test(FASTEST_TestOutput_t *out) {
    out->test_flags  |= FASTEST_ASSERT_EQ;
    out->exit_status |= add(2, 3) == 5 ? FASTEST_SUCCESS : FASTEST_ERROR_ASSERT;
}

FASTEST_CUSTOMTEST("custom test", FASTEST_TIME_NS | FASTEST_FAIL_ERROR, my_test, NULL);

int main(void) {
    FASTEST_QUICKTEST("Addition", add(2, 3) == 5,
        FASTEST_ASSERT_EQ | FASTEST_FAIL_ERROR | FASTEST_TIME_NS);
    return 0;
}
```

### Memory tracking

```c
#include "fastest/custom_tests.h"
#include "fastest/tests.h"
#include <stdlib.h>

void mem_test(FASTEST_TestOutput_t *out) {
    out->test_flags |= FASTEST_MEM_TRACK | FASTEST_FAIL_ERROR;

    void *ptr = malloc(64);
    out->allocation += 64;
    free(ptr);
    out->deallocation += 64;

    out->exit_status |= FASTEST_SUCCESS | FASTEST_DEFAULT_LOG;
}

FASTEST_CUSTOMTEST("mem/alloc-free", 0, mem_test, NULL);
```
